## Accessibility

### Form

Checkboxes must be used within a form element.

```javascript
<form>
  <Checkbox id="Primary-checked" label="Accept conditions" onChange={onChange} />
</form>
```

### Label

A label must be associated to the Checkbox.

If you don't want to display it, use the `hideLabel` prop, that will hide it properly, making it available for screen readers.  
Ensure that without displayed label, the context allows the user to understand easily the purpose of this Checkbox.

```diff
<form>
  <Checkbox
    id="Primary-checked"
    label="Accept conditions"
    onChange={onChange}
+   hideLabel
  />
</form>
```

## DO / Don't

### When to use Checkbox, when to use Radio

Don't use this component to give a choice between multiple elements, handling a unique checked state programmatically. Use Radio instead

```javascript
// DON'T ü§¢
<form>
  <Checkbox id="Mice" label="Mice" onChange={uncheckedOthers} />
  <Checkbox id="Dogs" label="Dogs" onChange={uncheckedOthers} checked />
  <Checkbox id="Cats" label="Cats" onChange={uncheckedOthers} />
</form>

// DO üòç
<form>
  <fieldset>
    <Radio id="Mice" label="Mice" value="mice" name="animal" />
    <Radio id="Dogs" label="Dogs" value="dogs" name="animal" checked />
    <Radio id="Cats" label="Cats" value="cats" name="animal" />
  </fieldset>
</form>
```

### When to use Primary and Secondary appearance

Use secondary appearance only for non important stuff

```javascript
// DON'T ü§¢
<form>
  <legend>My important stuff</legend>
  <Checkbox id="accept" label="Accept condition" appearance="secondary" />
</form>

// DO üòç
<form>
  <legend>My important stuff</legend>
  <Checkbox id="accept" label="Accept condition" />
</form>
```
